什么是Markdown

Markdown(以后简称MD) 是一种轻量级的「标记语言」，所以，他不是一个APP应用。MD的目标是实现「易读易写」
MD目标是成为一种适用于网络的书写语言。HTML是发布格式，MD是书写语言，MD并没有想去去取代HTML，而是通过对HTML中常用的文档书写内容进行重新定义，获得更好的写作体验。MD中未定义的HTML标签也可以写在MD文档中。

为何要是用Markdown

  1. 文本容易转换为HTML、PDF
  2. 关注于文本内容，而不是排版
  3. 简单易上手以及越来越多支持

工具与环境

　　越来越多的在线离线文本编辑器支持Markdown语法并支持实时预览。在线的有作业部落，简书，github等，离线的有（windows平台）MarkdownPAD，Atom的MDPT插件，为知笔记插件支持等。推荐使用Atom免费生成HTML，当然，如果只需要md格式文件，用什么都无所谓。

推荐两款初学和精深的使用工具。初学者：MarkdownPAD，内含类似于word的格式化工具，不用学习MD的语法。精深者适合atom，完全源码编写。

  • 文字修饰
    用两个 * 包含一段文本就是粗体的语法，用一个 * 包含一段文本就是斜体的语法，用两个 ~ 表示文本删除线

进阶使用

  • 嵌套序列
    实现嵌套序列只需要更改不同等级序列前面的空格数即可，空格数为2的倍数增长 

    * 一级序列  
     * 二级序列
       * 三级序列
    • 一级序列
      • 二级序列
        • 三级序列

    同理，有序序列一样

  • 单页面跳转
    使用链接实现单页面跳转有很多作用，比如制作目录，其原理就是链接，但是指向的是要找地方的id属性。MD有自动生成目录的命令[TOC]，但是亲测windows下不可用。所以只能使用单页面跳转来完成。 
    跳转起点：[跳转链接](#1)  
    跳转终点：<span id="1">跳转终点</span>
  • 可重用参考式链接
    链接分为行内连接（前面就是）和参考式链接。参考式链接相对于行内连接具有更易读和可重用的优势。对于图片也可以这样使用
    跳转到这个[注释][baidu]
    baidu: http://baidu.com "这是百度主页"（可以放在任意位置）
    跳转到这个注释

注意的点

  1. 两个空格作为回车。
  2. 大部分规则都适用，但是有些特异性的只有在某些地方才能适用。比如复选框还有emoj。emoj使用:joy:，显示为。emoj对照地址，实际引用地址
  3. 适用table请注意前面要有空行
  4. MD支持LaTex公式和流程图。LaTex公式符号对照表
  5. MD的空格。可以在中文输入法下切换全角 ( Shift + Space ) 空格代表一个汉字大小。也可以使用&emsp;&ensp;&nbsp;等
  6. MD很多情况不如人意，需要自己对HTML改造。比如a标签原页面打开，我们可以通过更改HTML属性 target="_blank" 设置其在新标签页打开。

大型项目的使用

　　以上所讲是语法及单个MD文档的书写，如果需要书写非常多的MD文档，按照上面的方法，只能一个个 SAVEAS HTML ，显得效率低下。所以下面讲解MD在大型文本编写中的应用。

  • 准备工作
    安装基于 Node.js 的 Gitbook 命令行工具，安装包位于svn的https://svnpd.ap.whchem.com/svn/ECS/EAD/trunk/Tools ，按照说明做即可。

    如图所示，每一个章节作为一个文件夹，而在外部会有一个 SUMMARY.md 作为目录文件，而在每个章节的内部会有 README.md 作为章节默认文档。

    按照上面所说语法编写即可，不再赘述。
